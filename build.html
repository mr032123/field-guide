Every hard question in the meeting is answered by one of these two rules.

Config is the exception: it's two columns (key, value) with one row per listed key — you type the real company names into the value column.

Words this guide uses (30-second glossary)

Sheet vs. Table: a sheet is the tab at the bottom of Excel; a Table is a formatted block of data on a sheet that has its own name — flows can only see Tables. · Flow: the automation you built in Copilot Studio (trigger at the top, then steps that run in order). · Trigger: the first box of a flow — what starts it (manual button, or a schedule). · Loop / "Apply to each": the container that repeats the steps inside it once per spreadsheet row. · Insert a field ("dynamic content"): in a flow, clicking into a box pops up a panel of values from earlier steps; clicking a field name (like prompt_text) places it in the box as a small colored tag meaning "the current row's value." · A run: one question asked once to one AI — one row of the Runs sheet. · n: how many times the same question is repeated; "n=8" = 8 runs of that question. · Browsing values: on = the Copilot agent has web search enabled; natural = ChatGPT asked with no instructions about the web (we record whether it searched); off = ChatGPT told not to search — a diagnostic subset that isolates what the model believes without the web. · Aided vs. unaided prompt: an aided prompt names a company in the question itself (so it cannot measure whether AI brings you up unprompted); an unaided prompt names no company.

1 · Question bank surgery

You have ~200 prompts per persona across the topics. Three operations turn them into an instrument: tag aided vs. unaided, add the decision tier (the aha source — your current bank barely has any), and freeze. Then expand it into the Runs sheet — one row per individual run.

1.1 · Extract your persona's prompts for the five topics
30 min · from your cleaned prompts file
›

Open your cleaned-prompts workbook. Click the small filter arrow in the Persona column header → untick Select All → tick only your persona → OK. Same on the Topic column header: tick only your five topics. Select the visible rows (click the first row, Shift-click the last), copy.

In measurement.xlsx, on the QuestionBank sheet: paste the prompt texts into the prompt_text column and the topics into the topic column (paste each column separately if your copy grabbed both — paste into a blank area first and move columns if needed; this is 5 minutes of tidying, don't aim for elegance). Fill qid with Q001, Q002…: type Q001 and Q002 in the first two cells, select both, then drag the small square at the selection's bottom-right corner (the fill handle) down the column — Excel continues the numbering.

Shortcut if the copy-paste fights you: attach BOTH workbooks to your assistant and say: "Copy all rows from the cleaned-prompts file for persona [name] and topics [list] into measurement.xlsx's QuestionBank sheet — prompt into prompt_text, topic into topic, sequential qids Q001 onward. Leave tier and prompt_class blank. Give me the file back."

1.2 · Tag aided vs unaided critical
15 min · one formula
›

Any prompt that names your parent company, product line, or a competitor is aided — it handed the answer a name and can never feed a presence or recommendation metric. Easiest path — attach the workbook to your assistant:

In QuestionBank, fill the prompt_class column: "aided" if the
prompt_text contains any of the six company names in Config's value
column, otherwise "unaided". Also fill the tier column for every row:
"decision" if the prompt forces choosing one option and rejecting
others; "comparison" if it weighs named options against each other;
"explanation" if it asks to explain or educate. Give me the file
back plus a count of rows per class and per tier.

Manual alternative: in Excel, use Find (Ctrl/Cmd-F) for each of the six names one at a time and type "aided" in prompt_class for every hit; everything left is "unaided". Either way: spot-check ten rows yourself — read the prompt, confirm the tag. The metrics' integrity rests on this column.

Expect the tier counts to come back almost entirely explanation/comparison with nearly no decision rows — that's the known gap, and the next step fixes it.

1.3 · Add the 20 decision prompts the aha source
15 min · copy from the Prompt library tab
›

Go to the Prompt library tab of this guide, copy your persona's entire block (it's one copy button), then attach the workbook to your assistant and paste: "Add these prompts to QuestionBank. They're formatted as 'ID | prompt text', grouped under topic headings. Give them qids in whatever numbering scheme your bank already uses (IDs are just labels — every later step selects prompts by the tier column, never by ID), the heading as topic, tier=decision, prompt_class=unaided. The PARAPHRASES section at the end: same treatment, but fill paraphrase_of with the ID each one rephrases (given in its label), and give them qids in your scheme. These decision prompts are what produce recommendation rate, who-wins, and the recurring objection language; the paraphrases measure whether results survive rewording — your stability number for "how robust is this."

1.4 · Select the run set and freeze pre-registration
30 min · then email it to yourself, dated
›

You will not run all ~200 prompts — you'll run a deliberate subset. Mark it by keeping only these in QuestionBank (or add an in_scope column):

  • All 20 decision prompts + 6 paraphrases (unaided) — the core.
  • 2 aided portrayal prompts per topic (10 total) — pick ones that name your product line and ask something substantive (from your existing bank). These measure how AI talks about you when asked directly.
  • 1 explanation prompt per topic (5 total) — for narrative color, and the tier where citation rate matters most.

That's 41 prompts. Freeze: set bank_frozen_date in Config, email the workbook to yourself with subject "bank frozen [date]". That email is your pre-registration receipt — the one-sentence answer to "you cherry-picked the questions."

1.5 · Expand the bank into the Runs sheet — one row per individual run
15 min · your assistant does it
›

Every single time a question will be asked — each repetition, on each surface, with each model — needs its own row in Runs, because the flows and your manual sessions work through Runs row by row. Attach the workbook to your assistant and paste:

From the QuestionBank sheet, generate the Runs sheet as one row per
run using these rules, and give me the workbook back:

- surface=copilot, model=gpt,    browsing=on, n=8  (run_index 1..8)
- surface=copilot, model=claude, browsing=on, n=8
- surface=chatgpt, model=gpt,  browsing=natural, n=5
- surface=chatgpt, model=gpt,  browsing=off,     n=3

browsing meanings: "on" = the Copilot agent has web search enabled;
"natural" = ChatGPT asked with no instructions about the web (it
searches if it wants to — we record whether it did in the searched
column); "off" = ChatGPT told not to search (diagnostic subset).

AIDED portrayal prompts:
- surface=copilot, model=gpt,    browsing=on, n=5
- surface=copilot, model=claude, browsing=on, n=5

EXPLANATION prompts:
- surface=copilot, model=gpt,    browsing=on, n=3
- surface=copilot, model=claude, browsing=on, n=3

run_id = R0001 onward. status = "pending". answer and run_date empty.
ALSO add a prompt_text column to Runs and fill it with each run's
full prompt text copied from QuestionBank (matched by qid) — the
flow reads the prompt from this column.
Report the total row count per surface/model so I can sanity-check.

Sanity-check the count report against this arithmetic: Copilot = 546 rows (26 decision × 8 × 2 models = 416, plus 10 aided × 5 × 2 = 100, plus 5 explanation × 3 × 2 = 30) — all automated. ChatGPT = 208 rows (26 × 5 natural + 26 × 3 web-off) — manual, ≈ 7 hours at ~2 min/run across three sessions. If that's too heavy, two sanctioned cuts: natural n=4 instead of 5 (saves 26 runs), and the web-off diagnostic for only 3 of 5 topics (saves ~24). Never cut Copilot — it's free labor. Trust arithmetic, not stamina.

Scaling rule — if your bank is bigger than the 41-prompt design Copilot runs everything: automated labor is cheap (watch for AI-credit warnings; if batches crawl, drop decision n from 8 to 6). ChatGPT stays fixed-size: only a pre-registered subset gets ChatGPT rows — for each topic, the first two decision-tier prompts in qid order. That is 2 × 5 topics × 8 runs = 80 manual runs, about 3 hours. Pick the subset by this rule NOW, before any results exist — choosing it after seeing data is cherry-picking. The ChatGPT-vs-Copilot comparison then uses only these matched prompts. If your Runs sheet already has ChatGPT rows for every decision prompt, paste this to your assistant with the workbook attached:
In the Runs sheet, keep rows with surface=chatgpt ONLY for this
subset: for each topic, the first two decision-tier prompts in qid
order (look up tier and topic in QuestionBank). Delete every other
surface=chatgpt row. Copilot rows are untouched. Report the subset
qids and the remaining chatgpt row count (should be ~80).
Why "natural" is the default condition The headline numbers must reflect what these tools actually do for a real client — and real answers use the web. The small web-off subset exists for one diagnostic reason: if a damaging narrative appears in web-off answers too, it's baked into the model itself (persists for years, unfixable by content); if it only appears with the web on, it's coming from specific pages you can go fix. That's what makes the prescriptions credible.
Note on the Copilot n=8 Not 10, deliberately: 8 keeps total agent calls near 500, inside plausible daily quotas, while still being a real sample. If batches run fast and error-free on Day 2, top up to 10 — add rows, never replace.

2 · Build the agents

Four agents in Copilot Studio (two panelist twins, one scorer, plus your proof-of-concept agent kept as a sandbox) and one Custom GPT in ChatGPT. The panelist twins are identical except for the model — that's what makes the GPT-vs-Claude comparison controlled.

2.1 · Panelist agent — GPT twin
20 min · same place you built the proof-of-concept flow agent
›

This agent's job: it plays the AI advisor serving your persona on the Copilot surface — the collection flow will feed it every question, one at a time, and log its answers. Go to copilotstudio.microsoft.com (where you built your proof-of-concept agent) → Agents in the left navigation → New agent. If it opens a chat asking you to describe the agent, look for a "Skip to configure" (or similar) link — you want the form, not the conversation. Name it Panelist GPT, and paste your persona's block below into the Instructions box.

Three settings to check on the agent's page: Model — leave the default GPT (this is the same dropdown where you spotted Claude; don't select it here, that's the twin). Knowledge — add nothing. Web search — find the toggle (usually under the agent's Settings, sometimes labeled "web search" or "allow the agent to use public web data") and turn it OFF — we want answers from the model's built-in knowledge only. Then click Publish (top right) — flows can only call published agents, as you learned in the proof-of-concept test.

Why there is no memory protocol here (there is one for ChatGPT in 2.4): every time the flow calls this agent, a completely new conversation starts — that is why each run's output shows a different conversationID. Runs cannot remember each other, so there is nothing to switch off. ChatGPT needed a protocol only because your personal account carries memory across chats. If you ever notice the same conversationID on two different runs, flag it — that would mean sessions are being reused, and the variance check in 3.2 would also catch it.

Naturalizing Jenny's existing prompts

The problem: her current bank appends "…to discuss with my advisor" to nearly everything. Real people don't ritually invoke their advisor — and the constant tag pushes every answer toward "ask your advisor" hedging, which suppresses the committed picks you're measuring. The rule: keep the advisor in at most one prompt in four, and only where validation IS the question. Elsewhere, keep her collaborative register through "we/our goals" framing, explainability asks, and peer/source-comparison instincts. Five before/afters from your actual bank:

BEFORE: For retirement planning, what bond ETFs offer steady income
at moderate risk that I could review with my advisor?
AFTER:  For retirement planning, which bond ETFs offer steady income
at moderate risk? I want to understand the trade-offs properly.

BEFORE: When I talk to my advisor, how should I weigh broad-market
ETFs versus picking individual stocks over the long term?
AFTER:  How should someone like me weigh broad-market ETFs against
picking individual stocks for the long term?

BEFORE: I'd like a safer, capital-preservation option to bring up
with my advisor. What kinds of ETFs fit that goal?
AFTER:  We want a safer, capital-preservation piece in our plan.
What kinds of ETFs fit that goal, and how would you compare them?

BEFORE: How does diversification cushion volatility? I want to
understand it well for conversations with my advisor.

it the way you would to a smart friend, so I can explain it to my
family.

KEEP (validation IS the point — advisor stays):

Troubleshooting — the ten ways this breaks

Everything below either happened during your proof-of-concept test or is the next-most-likely failure. Each entry: symptom → cause → fix. If something breaks that isn't here, describe it to your AI health build assistant with a screenshot — that's what it's for.

Flow error: "no dependent actions" / structural failure. An action isn't linked into the chain, a loop container is empty, or the trigger is an agent-trigger being run manually. Check the canvas is one connected vertical chain; re-add stray actions via the + between nodes; use a manual or recurrence trigger.
Agent gets no question / empty answers. The Message field on "Execute Agent and wait" is hidden under Advanced parameters → Show all. Click into Message and insert prompt_text from the pop-up panel. (Both loop actions hide their key fields this way — the answer column on Update a row too.)
Update a row fails. Key Column must be your ID column (exact spelling), Key Value: insert run_id from the List rows step — never conversationID. The file must be closed in desktop Excel while flows write. Locale field: en-US.
List rows returns nothing. Data isn't formatted as a named Table (Insert → Table, name it), or the Location/Library/File/Table dropdowns weren't picked from the dropdowns. Re-select all four.
Filter Query error: "Invalid filter clause: unsupported operation." The Excel connector accepts only ONE condition — any "and" fails (confirmed in this tenant). Keep status eq 'pending' as the sole filter and check surface/model with a Condition action inside the loop (step 3.1 has the exact fields).
Condition shows "Not specified" at runtime / phantom empty rows in the condition editor. The condition editor in this tenant saves malformed conditions (confirmed). Don't fight it — use Plan B in step 3.1: delete the Condition and switch to the filterkey column + single-clause filter. Remember to drag the three actions out of the True branch BEFORE deleting the Condition.
Flow "succeeds" but many rows never get answers. The Excel connector's default 256-row limit. Fix: "List rows" → ⋯ menu → Settings → Pagination On → Threshold 1000 (step 3.1 has it). Rerun — done rows are skipped, the missed ones get picked up.
Batches stall or rows stay pending. Rate limiting. Confirm the 30s Delay action is inside the loop; run again — the status filter makes reruns resume, not duplicate. If stalls persist, run smaller batches (filter by topic) and space them across the day.
The agent won't pick / hedges everything to "consult a professional." The instruction block lost its ALWAYS-answer + commit-to-ONE lines (usually a partial paste). Re-paste the full block, republish, reset affected rows to pending. Never fix this by rewriting bank prompts — the bank is frozen.
Copilot answers are word-for-word identical across runs. Caching — sampling is broken on that path. Verify you're asking via fresh flow runs (not one chat session), note it in Config, and treat Copilot n as reduced. ChatGPT runs are unaffected; lean on them for sampling claims if this hits.
ChatGPT browsing won't turn off. If "answer from your own knowledge only — do not search the web" still triggers search (links/citations appear), discard, retry once in a new chat; if persistent, note in Config that the web-off diagnostic is unavailable in your workspace and drop that subset — the natural runs are the headline data anyway.
